## 4. Standing code rules

Universal rules that apply across every product repo. Keep this list short and absolute.

1. **No `any` types.** Use proper interfaces and strict types. If `any` is unavoidable, comment why.
2. **No `console.log` in production code.** All logging must be structured or removed before commit.
3. **No silent catches.** Catch blocks either throw typed errors or return standardized error objects. A bare `try { ... } catch (e) {}` hides bugs and is forbidden.
4. **Number formatting safety.** Wrap `.toLocaleString()` calls with `Number()` to prevent runtime crashes on null/undefined.
5. **Additive-only migrations.** Database migrations are additive and non-destructive. Never drop columns or tables without explicit approval.
6. **API integrations.** Never use SDKs in server routes (Stripe, Twilio, etc.). Always use direct REST `fetch` calls.
7. **Secrets via a password manager only.** Never paste tokens or secrets in chat, write them to files, or commit them. Resolve secrets at use-time with your password-manager CLI (`op read "op://<vault>/<item>/<field>"` for 1Password, equivalent for Bitwarden / pass / etc.). The repo references env-var names, never values.

Add rules as you discover them; never delete one. When a rule changes, supersede it — see `memory-conventions/supersession-pattern.md`.

## 5. Brain conventions

Patterns that shape how every doc, log, and memory entry in this repo behaves. Each is codified as a ready-to-paste memory entry in `memory-conventions/`.

1. **4-Place Rule** — every meaningful change lands in four files: `Daily_Log.md`, `Change_Log.md`, `Decision_Log.md` (if a decision was made), `Feature_Inventory.md` (if feature status changed). See `memory-conventions/four-place-rule.md`.
2. **Supersession pattern** — when a memory entry, decision, or spec section is replaced, never delete the old version. Mark it superseded with a forward pointer. See `memory-conventions/supersession-pattern.md`.
3. **Cascade staleness** — docs that derive from upstream facts declare a `**Depends on:**` block. `bin/stale-audit.sh` surfaces dependents whose upstreams were recently touched. See `memory-conventions/cascade-staleness.md`.
4. **Per-lane Current_State** — each lane gets an auto-generated `Current_State.md` rolled up from its Decision / Daily / Feature files. Generated by `bin/lane-summary.sh`; never hand-edited. See `memory-conventions/lane-current-state.md`.

Add a convention when you've solved a real, repeating brain-rot problem and the fix is stack-agnostic.
