## YYYY-MM-DD — [name of your project or initiative]

One sentence on what this project is and why it exists. Then a sentence on current state (what's done, what's in flight, what's blocked). Reference any canonical doc if one exists.

**Why:** The motivation behind this work — a constraint, a deadline, a customer request, or a growth goal. Future-you (and future sessions) need this to make good judgment calls.

**How to apply:** What the agent should do differently because of this project — prioritize it, avoid conflicting changes, flag risks.

**Example:**

## 2026-06-01 — Online booking system launch

Building a customer-facing booking page so clients can self-schedule without calling the office. Currently in testing — form works, calendar sync is live, email confirmations not yet wired.

**Why:** Reducing inbound call volume is the #1 ops goal this quarter. The owner's time on the phone is 8–10 hours/week that could be automated.

**How to apply:** Any changes to the website should be checked against the booking flow. Do not modify the contact page without confirming it still routes to the booking system.
